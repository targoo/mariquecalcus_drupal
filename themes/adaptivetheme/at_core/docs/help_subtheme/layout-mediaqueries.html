Media Queries

This document covers how media queries are declared and used in this theme. This is not a guide to general use of media queries, it assumes you know the basics.

Three Breakpoints

By default all Adaptivetheme layouts use three media queries, or "breakpoints". These are declared as variables in each layouts _base.scss partial, i.e.:

$narrow      : "all and (min-width: 45em) and (max-width: 60em)";
$narrow-wide : "all and (min-width: 45em)";
$wide        : "all and (min-width: 60em)";

$narrow isolates styles withing a range - 45 to 60em.

$narrow-wide styles begin at 45em and continue to be used by all screen sizes over that width.

$wide starts at 60em and is used for all screens over that size.

Lets look at how that actually works in practice:

|<-- 45em         60em -->|
|<-- narrow ------------->|
|<-- narrow wide ------------------------------------------------------->
                          |<--- wide ----------------------------------->

Usage

The standard approach used in all default layouts is to wrap all declarations in a single media query for each breakpoint, rather than embedding media queries within the declarations themselves. This makes it a lot easier to build out the IE8 stylesheet without relying on additional SASS plugins.

Breakpoints are used in the main .scss file for your layout. For example lets look at the elf-layout-kit.scss file - we have removed declarations for clarity, what we are looking at is how the breakpoint varaibles are being used:

// Narrow
@media #{$narrow} {
}

// Narrow Wide
@media #{$narrow-wide} {
}

// Wide
@media #{$wide} {
}

If you read the section on the ELF you will see we use these breakpoints when importing ELF layout partials. If you take a look at the Minima layout included in the at_minimal sub-theme you will see we declare layout directly within these breakpoints rather than using @import (because that layout does not use the ELF). It's up to you.

SASS geeks will see that we are using the #{} interpolation syntax here to include the breakpoint variables as an "expression" in the @media directive.

Which Breakpoint to Use?

Approach it like this:

  1. To affect ONLY smaller screens like tablets, use the "narrow" breakpoint.
  2. To affect tablets AND wider screens use "narrow-wide"
  3. To affect ONLY wide screens use "wide".

A common mistake is to apply a snippet to "narrow-wide" and then try to override it in "wide". Instead move the narrow-wide snippet to the "narrow" breakpoint.

Why Ems?

Ems are the perfect compliment to a fluid grid approach in responsive design. They allow the layout to change depending on the users font-size setting - for example if the user needs to zoom the page the layout can react (change) to that input, for example, move a sidebar below the content to give longer line lengths (they are zooming the page to get bigger text, so we give them more room to read it).

This behavior happens automatically in Adaptivetheme layouts, as the font-size increases breakpoints will be trigged and the layout will change (in most modern browsers).

Ems give us truly responsive capabilities where the user takes back some control of what they are viewing - i.e. responding to the users needs, as opposed to them just viewing what we think they should.

Modifying Breakpoints

These breakpoints are a starting point, they may or may not suit your site (they are pretty good defaults for most). Consider your sites requirements, you may need to add new breakpoints, or change the existing breakpoints to suit your sites content.
