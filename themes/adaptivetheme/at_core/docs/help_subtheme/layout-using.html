Configuring Layouts in Appearance Settings

Adaptivetheme uses "layouts" to set the regions and CSS layout for a page. This can be for the default page and template suggestions.

Bascially it works like this: you choose default or template suggestion from the select list, then choose a layout, decide if you need to support IE8, then save the configuration. If you can do that, you can change your themes layout.

Default Page Layout

The first step is to configure the default page layout. This will re-write your themes page.html.twig file with new markup (twig) and inject any new regions into your themes info.yml file. Backups are always made before this happens and are stored in the /backup/ directory in your theme. You can switch off automatic backups in your themes settings if you prefer not to bother.

  1. Choose "Default" from the Page Type options.
  2. Select a layout.
  3. Configure Internet Explorer Options.
  4. Save the configuration.

Changing the Default Page Layout

At any time you can change the default page layout simply by repeating the process above.

NOTE: if you have existing template suggestions and now want to change the default to a different plugin, then you must delete all the template suggestions first. Then configure your new default layout, then recreate the template suggestion using layouts from the new plugin.

Help! My Regions are not showing up in block config!

Clear the cache admin/config/development/performance. It's really important especially if you changed your layouts code, or chose a default page layout from a new plugin. You must clear the cache so Drupal's block system can "see" and use the new regions.

Template Suggestions

Drupal supports something called "template suggestions". These override the default template when invoked, usually when a particular path is visited in the site - for example the front page, user page, forum pages and so on. Template suggestions do not exists by default, you have to create them and this theme provides a very simple way of generating template suggestions.

Creating template suggestions is strait forward:

  1. Choose "Template Suggestion" from the Page Type options.
  2. Enter the template suggestion.
  3. Select a layout - this must be from the same plugin as the default layout.
  4. Configure Internet Explorer Options.
  5. Save the configuration.

Like the default layout you can change a template suggestions layout simply by repeating the process.

Available Suggestions

If you don't know what suggestions are available use the Show Page Suggestions feature, which you will find under the Devel tab. Now when you visit a page you will see 3 or 4 suggestions printed in your themes "messages" area. The one at the top is always the least specific and the one at the bottom the most specific (more specific suggestions override less specific.) For example when you visit the user page for user 1 you should get this:

  • page__user - applies to http://www.example.com/user
  • page__user__% - applies to http://www.example.com/user/[n]
  • page__user__1 - applies only to http://www.example.com/user/1

You can see that page__user is the least specific, so if you want the suggestion to apply to ALL user pages, that is the one to use, if you only want it to apply to user 1's user page, use page__user__1. This holds true for all page template suggestions. The odd one in the middle with the % (percentage) symbol will match any number.

Another quick example would be for node pages (not node templates, they have their own template suggestions, this is just for the page templates):

  • page__node - matches all nodes.
  • page__node__article - matches all "article type" nodes (see below).
  • page__node__1 - matches only node/1.

Here are a few other common suggestions you can use:

  • page__front - site homepage, i.e. http://www.example.com/
  • page__contact - contact page, i.e. http://www.example.com/contact (using Drupal core Contact module)
  • page__forum - all forum pages, i.e. http://www.example.com/forum (using Drupal core Forum module)
  • page__tracker - tracker pages, i.e. http://www.example.com/tracker (using Drupal core Tracker module)

The Views module adds a template suggestion for all Views pages and can be seen when using the Show Page Suggestions feature.

Content Type Page Suggestions

Adaptivetheme adds a template suggestion for each content type. If you have a content type of "article" the suggestion will be page__node__article. You can find these the same way as other suggestions by using the Show Page Suggestions feature in Devel settings.
