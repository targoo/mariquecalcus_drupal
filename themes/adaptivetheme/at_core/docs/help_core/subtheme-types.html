Starterkits

Adaptivetheme supports three Starterkit types - "Standard", "Minimal" and "Skin".

Standard

The standard type is great for site builders and those who do not want to build layout. Using the Standard starterkit you can build your entire site and then hand off to the themer to add the design styles.

For themers this is also a great start - you get a very robust layouts, extra tools like SASS support and can build out themes very quickly.

The Standard starterkit ships with three layouts that all have 22 regions. You can configure many different sites and layouts merely by placing blocks in regions and using Drupal cores excellent block visibility settings. Adpativethemes will also provide a Layouts repository on Github where you will be able to download and install new layouts for your themes - this is currently in development and we're hoping it becomes a large repository of hundreds of layouts.

All themes are supported by AT Cores "Libraries" feature - this allows you to drop any "library" (a collection of JavaScript and/or CSS files) into a folder call "/libraries/" in the root of your sub-theme and they will immediately be available in your theme settings where you can toggle them on and off. This makes adding things like jQuery plugins trivial. See help section for "Libraries" in your sub-theme for more details, or the README in the /libraries/ folder of your theme. Note that this feature includes the ability to toggle Drupal core libraries on and off also.

Key Features

  • Three advanced Layouts (easily add more from the Layout library or build your own)
  • Responsive in mobile and small screens
  • SASS Support (normal CSS files also)
  • Layouts built with Susy
  • Extra SASS mixins e.g. visually-hidden, image-replace
  • Libraries support

Standard type is really designed for rapid site and theme development, if you're in a hurry then this is a good choice.

Minimal

As per the name, the Minimal starterkit has fewer features and will suite Front End Developers. The major difference is that it ships with just one simple layout, so is really designed for those wanting to build their own Layout right from the start.

The included layout is built with Susy Next (Susy version 2), however it's so basic you can easily switch to using your own bespoke layout CSS, or use your usual SASS framework such as Zen Grids, Zurb Foundation, Twitter Bootstrap etc.

Key Features

  • One responsive layout
  • SASS Support (normal CSS files also)
  • Layout built with Susy
  • Extra SASS mixins e.g. visually-hidden, image-replace
  • Libraries support

Skin

Skins are intended to be sub sub-themes. They only have one template - a page.html.twig since this is very important in Adaptivetheme for supporting the Layouts Extension. Sub-themes can have their own layouts and you can generate Layouts and template suggestions just for your skin - bascially they are fully fledged themes like any other, but we strip out some stuff that's not needed since its inherited from it's base theme.

The original intention of Skins was to be a replacement for Footheme, i.e. to be an easy way to sub-theme existing Color module enabled sub-themes. At this time Skins are still in development and not well tested.

First you must generate (or manually create) and theme based on the Standard or Minimal starterkit, then you will get the option to select the Skin type.

When you select "Skin" the generator wizard will show you a list of available base themes.

Key Features

  • Inherit their base themed settings and template overrides
  • SASS Support (normal CSS files also)
  • Libraries support

Important! When generating Skin type themes you must update the info.yml file - specifically the stylesheets and scripts information.
