Updating Skin Info Files

Skin generated sub-themes need to have their info.yml files checked and updated. Specifically you need to look at the stylesheet and script declarations.

Note that the Generator Wizard may take care of these clean-ups in the future, right now these are the only caveats to creating new "Skin type" sub-themes.

Stylesheets

By default your Skin generated sub-themes info.yml file will have a direct copy of it's base themes stylesheet declarations, which will probably look something like this:

  stylesheets:
    all:
      - assets/css/layout.css
      - assets/css/screen.css

If you leave it like this your Skin theme will override the base themes styles - entirely. This is Drupal core behaviour and is well documented here: Overriding style sheets from modules and base themes.

If you do not want this to happen, you will need to change the name of the stylesheets.

To Update the Stylesheet declarations:

  1. Locate the info.yml file in your themes root, always named [themeName].info.yml
  2. Change or remove the declarations you want to affect. For example you probably want to remove assets/css/layout.css so your Skin theme gets all the base themes layout.
  3. Next, locate the actual CSS files in assets/css/ and update the name changes there also, or remove files you are not using.
  4. Clear the sites cache

Scripts

All script declarations have been copied over as well, however its unlikely you would want to declare these - you can simply remove all script declarations as your Skin them will inherit any scripts automatically from the base theme.
